# Práctica - Módulo 16

## Objetivo práctico

Diseñar una arquitectura on premise segura para incorporar IA en una empresa con bases Oracle transaccionales, minimizando fuga de información y evitando acciones destructivas.

## Modalidad sugerida

Laboratorio guiado en parejas o individual.

## Actividades

- Actividad 1: identificar riesgos del escenario actual
- Actividad 2: proponer los componentes de la arquitectura objetivo
- Actividad 3: definir controles de identidad, secretos y acceso a datos
- Actividad 4: diseñar flujos permitidos y flujos prohibidos
- Actividad 5: producir una propuesta técnica documentada

## Material de esta sección

- [Ejercicio 01 - Diseñar una arquitectura on premise segura para usar IA](./ejercicio-01-arquitectura-onpremise-segura-ia.md)

## Criterio de éxito

Al terminar la práctica, el estudiante debe poder:

- reconocer los riesgos del escenario actual
- proponer una arquitectura segura por capas
- justificar controles concretos sobre accesos, datos y agentes
- explicar cómo se reduce el riesgo de fuga o borrado
