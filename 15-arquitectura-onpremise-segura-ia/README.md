# Módulo 16 - Arquitectura on premise segura para IA

## Objetivo del módulo

Entender qué componentes y controles deben incorporarse en una arquitectura on premise para usar IA de forma segura en una empresa que trabaja con bases de datos internas y procesos críticos.

## Enfoque del módulo

Este módulo combina una dosis corta de teoría con un ejercicio de diseño aplicado. El foco no está en instalar una herramienta puntual, sino en diseñar una arquitectura segura y realista para reducir fuga de información, limitar acciones destructivas y controlar el acceso de agentes y asistentes de IA a sistemas sensibles.

## Resultados de aprendizaje

Al finalizar este módulo, el estudiante podrá:

- identificar riesgos de seguridad al incorporar IA en un entorno on premise
- proponer componentes mínimos de una arquitectura segura para IA
- separar zonas de acceso, inferencia, datos y observabilidad
- definir controles de identidad, secretos, auditoría y protección de datos
- diseñar una propuesta técnica razonable para una empresa con bases Oracle on premise

- 5 min: revisión final

## Mensaje clave

Implementar IA en una empresa on premise no consiste solo en instalar un modelo o un agente. Requiere rediseñar cómo se accede a datos, cómo se autentican usuarios y servicios, cómo se registran las acciones y cómo se evita que la IA tenga más permisos de los que realmente necesita.
