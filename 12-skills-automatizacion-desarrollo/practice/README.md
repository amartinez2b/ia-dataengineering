# Práctica - Módulo 12

## Objetivo práctico

Instalar y usar capacidades reutilizables del ecosistema de agentes para resolver tareas concretas del curso.

La práctica principal se centra en la skill `docx` para generar un documento Word con la documentación técnica del proyecto de datos construido en los temas 11 y 12.

Como ejercicios complementarios, también se exploran la instalación y uso del plugin `Chrome` en Codex, la skill `prompt-master` en Claude Code y Codex, y la skill `architecture-diagram-generator` para producir diagramas técnicos en HTML.

Los ejercicios complementarios pueden trabajarse como extensión del laboratorio principal o como práctica adicional fuera de la agenda base del módulo.

## Modalidad sugerida

Laboratorio guiado en parejas o individual.

## Actividades

- Actividad 1: instalar la skill `docx` en Codex
- Actividad 2: preparar la información técnica del proyecto
- Actividad 3: usar la skill `docx` para generar un `.docx`
- Actividad 4: repetir el ejercicio en OpenCode con instalación local de la skill
- Actividad 5: instalar el plugin `Chrome` en Codex
- Actividad 6: usar `@chrome` para resumir un sitio web
- Actividad 7: instalar la skill `prompt-master` en Claude Code
- Actividad 8: instalar la skill `prompt-master` en Codex
- Actividad 9: usar `prompt-master` para mejorar prompts del curso
- Actividad 10: instalar la skill `architecture-diagram-generator`
- Actividad 11: describir la arquitectura del ciclo de vida de datos
- Actividad 12: generar el diagrama técnico en HTML
- Actividad 13: iterar y validar el resultado visual
- Actividad 14: revisar los resultados y validar calidad de entregables

## Criterio de éxito

Al terminar la práctica, el estudiante debe poder:

- instalar una skill especializada en Claude Code
- instalar una skill especializada en OpenCode
- instalar un plugin en Codex
- instalar una skill reutilizable en Codex
- instalar una skill pública orientada a diagramación técnica
- usar la skill `docx` para crear un documento Word
- usar `@chrome` para ejecutar una tarea web asistida
- generar un diagrama técnico en HTML con una skill especializada
- usar `prompt-master` para mejorar prompts de trabajo
- documentar técnicamente el pipeline y el DAG del proyecto
- obtener un entregable profesional reutilizable
