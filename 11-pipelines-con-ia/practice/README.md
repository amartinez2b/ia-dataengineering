# Práctica - Módulo 12

## Objetivo práctico

Construir un DAG de Airflow que orqueste todo el flujo de datos desarrollado en el módulo 11, usando agentes de IA durante todo el proceso.

## Modalidad sugerida

Laboratorio guiado en parejas o individual.

## Actividades

- Actividad 1: analizar el pipeline existente del tema 11
- Actividad 2: diseñar las tareas del DAG
- Actividad 3: generar el DAG con ayuda de agentes de IA
- Actividad 4: revisar dependencias, orden y validaciones
- Actividad 5: dejar el flujo listo para ejecución o revisión técnica

## Material de esta sección

- [Ejercicio 01 - Orquestar el pipeline PySpark con un DAG de Airflow usando agentes](./ejercicio-01-dag-airflow-con-agentes.md)

## Criterio de éxito

Al terminar la práctica, el estudiante debe poder:

- representar un pipeline de datos como un DAG
- modelar dependencias correctas entre tareas
- usar agentes de IA para acelerar el desarrollo del DAG
- explicar cómo el DAG orquesta `bronze`, `silver` y `gold`
