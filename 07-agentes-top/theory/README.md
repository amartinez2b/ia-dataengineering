# Teoría - Módulo 07

## 1. ¿Por qué comparar varios agentes?

Aunque muchos agentes parecen similares, en la práctica cambian bastante en:

- instalación
- autenticación
- entorno de trabajo
- integración con herramientas
- experiencia en terminal o editor

Por eso conviene conocer más de uno.

## 2. Qué vamos a comparar en este módulo

En este módulo nos enfocamos en:

- Codex
- Gemini CLI
- OpenCode

Los tres permiten trabajar desde terminal, pero su experiencia y ecosistema son distintos.

## 3. Codex

Codex está orientado a trabajo agentic con foco fuerte en código, repositorios, terminal y MCP.

### En qué destaca

- trabajo sobre repositorios
- integración con flujos modernos de agentes
- buenas capacidades de coding
- uso desde CLI e integración en entornos más amplios

## 4. Gemini CLI

Gemini CLI lleva el ecosistema Gemini al terminal y permite trabajar con prompts, herramientas y servidores MCP desde línea de comandos.

### En qué destaca

- buena integración con ecosistema Google
- acceso mediante API key de AI Studio
- enfoque terminal-first
- compatibilidad con MCP

## 5. OpenCode

OpenCode está orientado a agentes de código con experiencia de terminal y automatización sobre repositorios y flujos de trabajo.

### En qué destaca

- experiencia orientada a repositorios
- trabajo con automatización y GitHub
- uso agentic desde CLI

## 6. Qué aprender de esta comparación

Más allá de instalar tres herramientas, la idea del módulo es entender:

- qué tan fácil es autenticarlas
- qué tan claro es el flujo de instalación
- qué tan usable es cada CLI
- en qué contexto una puede ser mejor que otra

## 7. Idea clave para llevarse

Un agente no se elige solo por marca. También importa:

- facilidad de instalación
- modelo disponible
- costo y API key
- entorno técnico del equipo
- integración con el flujo de desarrollo
