# Módulo 11 - Proyecto práctico 2: desarrollo PySpark

## Objetivo del módulo

Desarrollar un flujo de ingeniería de datos con ayuda de agentes de IA sobre un proyecto real, implementando procesos `raw -> bronze -> silver -> gold` con Python, PySpark y notebooks.

## Enfoque del módulo

Este módulo es principalmente práctico. La teoría solo entrega el marco mínimo para entender la arquitectura por capas, el rol de los notebooks y cómo coordinar el trabajo con agentes de IA dentro de un repositorio real.

## Resultados de aprendizaje

Al finalizar este módulo, el estudiante podrá:

- copiar un repositorio real a su cuenta personal de GitHub
- crear y usar una rama `dev` para desarrollar cambios
- usar agentes de IA para analizar un proyecto de datos existente
- implementar ingestas de `raw` a `bronze`
- construir limpiezas de `bronze` a `silver`
- consolidar datos de negocio en una capa `gold`

## Agenda sugerida para 1.5 horas

- 15 min: arquitectura `raw -> bronze -> silver -> gold`
- 15 min: análisis inicial del repositorio con agentes
- 20 min: diseño de las ingestas y limpieza por tabla
- 30 min: desarrollo asistido de scripts y notebooks
- 10 min: validación de resultados y cierre

## Mensaje clave

En un proyecto PySpark real, el valor de usar agentes de IA no está solo en generar código rápido. Está en ayudar a entender el repositorio, proponer estructuras coherentes, acelerar transformaciones repetitivas y apoyar la validación del pipeline sin perder control humano del diseño y la calidad.
