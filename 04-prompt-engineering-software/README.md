# Módulo 04 - Ingeniería de prompts para desarrollo de software

## Objetivo del módulo

Aprender a diseñar prompts claros, estructurados y efectivos para obtener mejores resultados al usar LLMs en tareas de desarrollo de software, especialmente en Python, PySpark y trabajo técnico relacionado con ingeniería de datos.

## Enfoque del módulo

Este módulo es altamente práctico. La teoría sirve como marco para entender por qué un prompt funciona mejor que otro, pero la meta principal es que el estudiante sepa iterar prompts y mejorar la calidad del código generado por un asistente de IA.

## Resultados de aprendizaje

Al finalizar este módulo, el estudiante podrá:

- identificar los componentes de un buen prompt
- estructurar prompts con objetivo, contexto y restricciones
- mejorar prompts ambiguos o débiles
- pedir código Python y PySpark de forma más precisa
- iterar prompts hasta obtener una respuesta más útil y confiable

## Agenda sugerida para 1.5 horas

- 20 min: qué hace bueno o malo a un prompt
- 20 min: componentes y patrones útiles
- 15 min: errores frecuentes al pedir código
- 25 min: ejercicios de iteración de prompts
- 10 min: cierre y conclusiones

## Mensaje clave

Un buen prompt no es magia. Es una especificación técnica escrita en lenguaje natural. Cuanto mejor definimos contexto, objetivo, restricciones y formato esperado, mejor suele ser el resultado.
