# Ejercicio 01 - Mejorar prompts débiles

## Objetivo

Aprender a transformar prompts ambiguos en instrucciones más claras y efectivas, usando Markdown como estructura base.

## Duración sugerida

15 a 20 minutos

## Actividad

Toma los siguientes prompts débiles y mejóralos.

### Prompt débil 1

```text
Explícame este código.
```

### Prompt débil 2

```text
Hazme un script para datos.
```

### Prompt débil 3

```text
Corrige este error.
```

## Instrucciones

Para cada prompt:

1. identifica qué información falta
2. reescríbelo usando Markdown
3. agrega rol, objetivo, contexto, restricciones y formato
4. incluye un ejemplo que ayude a la IA a entender mejor la salida esperada
5. escribe una nueva versión mejorada

### Importancia de usar Markdown y ejemplos

En este ejercicio no basta con mejorar el contenido. También debes mejorar la **estructura** del prompt.

Markdown ayuda a:

- separar ideas
- ordenar instrucciones
- reducir ambigüedad
- hacer más fácil la lectura del prompt

Además, incluir ejemplos ayuda a:

- mostrar el formato de salida esperado
- reducir variación innecesaria
- obtener respuestas más consistentes

Si esperas una tabla, un JSON, una lista o una explicación estructurada, debes mostrar un ejemplo pequeño dentro del prompt.

## Formato obligatorio

Cada prompt mejorado debe escribirse en Markdown y usar al menos:

- un título o sección
- una lista de restricciones
- una sección de formato esperado
- un ejemplo de salida esperada

## Entregable

El estudiante debe entregar:

- versión original
- problemas detectados
- versión mejorada en Markdown
- ejemplo incluido dentro del prompt

## Preguntas de reflexión

- ¿Qué parte fue más importante al mejorar el prompt?
- ¿Qué cambió al agregar contexto o restricciones?

## Aprendizaje esperado

El estudiante debería concluir que un prompt técnico útil casi siempre necesita más que una sola instrucción genérica, y que Markdown más ejemplos concretos ayudan a organizar mejor lo que queremos pedir y a obtener respuestas más consistentes.
